正在做日綜／日本搞笑藝人字幕的日文→台灣繁中翻譯。目標不是正式學術翻譯，而是讓中文字幕自然、好讀、辨識度高，同時不要亂創台式譯名。核心目標是：讓中文觀眾盡可能接收到日本觀眾看到藝名／團名時能感受到的資訊量。

## 檔案結構：talents / others

`fixed_glossary.json` 分兩大類：

```jsonc
MappingBlock = { "jp": [str, ...], "zh": str }

{
  "talents": [
    {
      "group": MappingBlock,            // 可選：組合名；solo 藝人省略
      "members": [MappingBlock, ...]    // 至少一名成員
    }
  ],
  "others": [MappingBlock, ...]         // 節目／單元／品牌／術語／作品等
}
````

* `talents`：搞笑藝人。組合用一個 unit 包起來，`group` 放組合名，`members` 放成員。
* `others`：節目名、單元名、品牌、平台、搞笑術語、作品名等非人物專名。
* 把組合和成員包在一起，是為了解決裸成員名模糊問題，例如 `きむ`、`リリー`、`ガク`、`ノブ`、`せいや`。命中任一成員或組合名時，整個 unit 可以一起餵給 pre-pass，讓模型知道上下文。

## jp[] 使用原則

`jp[]` 只放組合／人物最原始、正確的 source 來源拼寫。

以下變體由 `services\gemini\fixed_glossary.py <_normalize_jp>` 處理，不要重複收進 `jp[]`：

* NFKC：全形→半形、相容字
* 片假名 → 平假名
* 去除長音符、波浪、中點、空白、Tab、換行、常見標點
* casefold：大小寫折疊

好例：

```jsonc
{ "jp": ["熊元プロレス"], "zh": "熊元摔角" }
{ "jp": ["ほんこん"], "zh": "Honkon" }
```

壞例：

```jsonc
{ "jp": ["熊元プロレス", "熊元 プロレス", "熊元　プロレス"], "zh": "熊元摔角" }
{ "jp": ["ほんこん", "ホンコン"], "zh": "Honkon" }
{ "jp": ["ホンコン", "香港"], "zh": "Honkon" } // 香港可能真的是地名
```

ASR 相似音或嚴重走鐘，不靠 `jp[]` 硬塞猜測，交給 full 模式與 LLM 根據音訊／畫面確認。

## 專名翻譯總原則

專名包含：節目名、單元名、搞笑團體名、藝人名、藝名、作品名、品牌名、梗名。

整體優先順序：命中第一個適用者即停。

### 1. 有穩定台灣／中文譯名就用既有譯名

只在能確認常見、穩定、官方或中文圈已廣泛使用時採用。不要自己發明看起來很像台灣節目名的創意譯名。

| ネプチューン         | 海王星   |

### 2. 人名／藝名可回推漢字時，要看該漢字是不是名稱核心

若假名只是本名、姓氏、名字的假名表記，或中文圈已有穩定漢字用法，可用繁中漢字。

| かが屋     | 加賀屋    |

### 4. 來源可解析的名稱，優先轉譯「名稱結構」與資訊量

如果團名、藝名或節目名的來源可解析，優先保留原名的命名結構，讓中文觀眾也能看出原本的梗、語意、漢字核心、成員姓名、地名或外來語成分。

這不是亂創譯名，而是把日本觀眾看得懂的命名資訊，轉成中文觀眾也能理解的形式。

處理方式不一定是全中文，也不一定是全羅馬字。可以依原名結構選擇：

- **中文化**：來源語意明確，翻成中文後仍像專名。
- **半翻半保留外來語**：原名本身是「中文可理解成分 + 外來語／英文」的混成。
- **保留漢字核心 + 羅馬字／英文**：原名有姓名、地名、漢字來源，但部分成分不適合翻中文。
- **回推姓名／地名／典故漢字**：來源明確且是名稱核心時，不要只羅馬字化。

判斷重點：

- 中文觀眾是否能從譯名看出原名的命名意圖。
- 翻完是否仍像專名，而不是普通句子、商品名、物品名或解釋文。
- 是否保留原名的結構比例，而不是一律全中文或全英文。
- 若只羅馬字化會讓中文觀眾完全看不出來源，應優先考慮中文化或半中文化。
- 外來語成分不是一律保留英文，也不是一律翻中文。若中文裡已有穩定、自然、且不失專名感的對應詞，可以中文化；若翻成中文會太解釋、太抽象、太普通，或失去原本外來語質感，則保留英文／羅馬字。

例：

翻中文或羅馬字都優先於保留假名；對看中文字幕、不懂日文的觀眾，羅馬字／中文的辨識度一律高於假名。唯一例外是名稱高度依賴日文文字視覺（字形）雙關、羅馬字化會直接毀掉該梗時才保留原文。單純語音雙關不算（羅馬字已保留讀音），辨識度較低也不算；其餘一律走 tier 4 羅馬字，或前面 tier 的中文／漢字回收。

```text

ケムリ
すがちゃん
や団
なえなの
ほんこん
かまいガチ
```

## 專名翻譯和一般字幕翻譯要分開

一般字幕句子中，不要擅自補「我／你／他／她／我們／大家」等日文省略主詞。

但節目名、藝名、梗名是固定名詞，可以為了保留語氣做固定化翻譯。

例：

| 類型   | 日文          | 繁中             |
| ---- | ----------- | -------------- |
| 一般句子 | 知らんけど       | 不一定要補成「我是不知道啦」 |
| 節目名  | かまいたちの知らんけど | 鎌鼬的我是不知道啦      |

## 不要做的事

* 不要把片假名藝名直接照意思硬翻。

  * 壞：`松井ケムリ` → `松井煙`
  * 好：`松井ケムリ` → `松井Kemuri`

* 不要擅自幫假名人名補漢字。

  * 壞：`きょん` → `恭`
  * 好：`きょん` → `Kyon`

* 不要把節目名翻成很像台灣節目的創意標題。

  * 壞：`かまいガチ` → `鐮鼬真挑戰`
  * 好：`かまいガチ` → `鎌鼬來真的`

* 不要把搞笑團體名硬翻成普通名詞。

  * 壞：`モグライダー` → `鼴鼠騎士`
  * 好：`モグライダー` → `Mogu Rider`

* 不要把字面有意思的藝名翻成普通句子。

  * 壞：`おいでやす小田` → `歡迎光臨小田`
  * 好：`おいでやす小田` → `Oideyasu小田`

* 不要把暱稱化藝名翻成自創中文暱稱。

  * 壞：`おたけ` → `小武` / `阿武`
  * 好：`おたけ` → `Otake`

* 不要在中文字幕裡大量留下日文假名。

  * 翻成中文後仍像專名，才中文化。
  * 否則優先羅馬字／常見英文表記。
  * 假名僅限日文文字視覺雙關才保留，語音雙關與辨識度都不算，其餘一律羅馬字／常見英文表記。

* 不要把「組合＋成員」拼成一條 `jp` 別名。

  * 組合關係用 `talents` 結構表達。
